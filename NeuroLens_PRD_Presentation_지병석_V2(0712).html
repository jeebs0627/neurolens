Part 5 · Intelligence Requirements — 제품의 심장

자동화 · 예측 · 시뮬레이션 · 최적화, 하나의 폐루프

기능 나열이 아니라, 네 가지 지능화가 동시에 작동해야 반복 매출과 데이터 해자가 만들어집니다. 각 항목마다 정량 성공 기준을 정의했습니다.

⚙️

자동화 AUTOMATION

측정 준비 → 위험군 분류 → 후속 행동 → 재측정까지 운영자 수작업 최소화. 일반군은 Agent로, 주의군은 관리자 확인으로, 고위험은 SOP에 따라 전문상담으로 자동 분기.

리포트 생성 60초 이내 자동 처리율 90%+ 관리자 1명 = 500명 캠페인 수작업 엑셀 제거
📈

예측 PREDICTION

단일 점수를 넘어 현재 위험도 + 악화 가능성 + 개입 필요도를 추정. 확정값 대신 위험구간 + 신뢰도·불확실성을 함께 표시하고, "우울증이다"가 아닌 "추가 확인이 필요한 위험 신호"로 제시.

AUC 0.819 기준선 유지 Recall 82.9% NPV 92% 드리프트 월 1회 모니터링
🧪

시뮬레이션 SIMULATION

개입 전 행동 시나리오와 예상 효과를 비교. "2주 루틴 수행 시 예상 변화 범위", "주 1회 vs 월 1회 재측정 민감도", 기관 운영부하·매출 시뮬레이터. 인과를 단정하지 않는 범위형 표시, 의료 치료효과는 시뮬레이션하지 않음.

가정·데이터 버전 100% 표시 3개 시나리오 5분 내 비교 오차 분기별 검토
🎛️

최적화 OPTIMIZATION

개인에겐 최적의 다음 행동을, 기관엔 효율적 자원 배분을 추천. 안전 규칙 기반 → 세그먼트 휴리스틱 → A/B 테스트 → Contextual Bandit 순으로 단계적 고도화.

Agent 4주 유지율 30%+ 콘텐츠 완료율 40%+ 재측정 전환율 35%+ 위험군 검토시간 −50%

안전·개인정보 원칙을 제품 요구사항으로 내장하고, 범위 제외를 명시하며, 리스크마다 완화책을 짝지었습니다.

안전·개인정보·윤리 — 제품에 내장
  • 미성년자 동의·보호자/기관 절차를 채널별 정의
  • 원본 영상 저장은 기본값 아님 — 필요 시 별도 동의
  • 고위험 신호는 자동 진단이 아닌 "추가 확인 필요" + 전문채널 안내
  • 자해·위기 표현 감지 시 즉시 중단·전문기관 안내·관리자 SOP
  • 집단 리포트는 최소 인원 기준 이하 비공개
  • 사용자 데이터 열람·정정·삭제 요청권 보장
범위 제외 — 이번 단계에서 약속하지 않는 것
  • 정신질환 확진 또는 치료 효과 보장
  • 임상적 응급판단의 완전 자동화
  • 외부재현 전 성능의 의료현장 일반화
  • 초기부터 모든 연령·질환·기기 지원
  • SaMD 매출의 Base Case 포함
  • 완전 자율형 Agent의 상담사 대체 주장

Part 17 · Final Success Criteria

"완성"의 기준은 기능 수가 아니라 동시 작동입니다

네 가지가 동시에 작동하고, 그것이 기관 라이선스 · Agent 구독 · API 사용료라는 실제 반복 매출과 데이터 해자로 연결되는 것 — 이것이 Seed 투자유치 가능성을 판단하는 최종 기준입니다.

1자동화

500명 단위 캠페인이 수작업 없이 운영된다

2예측

현재 위험과 변화 가능성을 불확실성과 함께 설명한다

3시뮬레이션

개입·운영·가격 시나리오를 실행 전에 비교한다

4최적화

반복 데이터로 다음 행동과 자원 배분이 점점 더 좋아진다

시선 추적으로 '객관적 생체 신호'를 읽어냅니다.
